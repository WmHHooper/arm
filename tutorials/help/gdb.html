GDB Quick Reference

gdb is the GNU Debugger. A complete online manual is available on the web for the latest version. Below is a list of the gdb commands and arguments used in these tutorials, with a brief explanation of each. The gdb command is the first word; the letters of the word in brackets may be omitted. Arguments to the command are shown in italics.

Command

Explanation

h[elp]

list a summary of gdb command categories, and explain how to get more help.

br[eak] Address set a breakpoint at Address. Address may be a number (e.g., 0x02000B7C) or a label (e.g., main).

r[un]

c[ontinue]

Load and run the program. After a break, continue resumes the program, while run starts it over. Once the program starts or resumes, gdb loses control until the program reaches a breakpoint, quits, crashes, or is interrupted by the user (e.g. by pressing Fn-C).

si N

step N

Step forward through N consecutive machine Instructions. If omitted, N defaults to 1. For programs written in C/++ and compiled with gcc -g, step steps through one line of source code.

p[rint] /fmtÂ expression

display the value of expression, using /fmt. Common expressions are names of variables and functions, register values $r0..$r12, $sp, $lr, $pc, $s0..$s31, and $d0..$d15. Common formats are: /d for decimal, /x for hexadecimal, /s for string, and /i for instruction. More elaborate formats are given in the references.

x  /fmt  Address

display memory, formatted according to /fmt, beginning at Address. /fmt has the same options as the print command.

disp[lay] /fmt expression

undisp[lay] Number

add expression and /fmt to the display list. Items on the list are displayed after each break. /fmt has the same options as in the print command. undisplay removes item Number from the display list; if Number is omitted, the display list is cleared.

disas[semble] Address

disassemble instructions. The optional Address allows you to dissassemble from some other location than the current PC; otherwise, it will disassemble the current function

info option

displays misc info about the debugging environment. Some useful options are: stack, locals, and frame.

quit

exit gdb, killing the program if it is running.
