Installing Software

Before you begin to program the , you will need to install some software on your own computer:

  1. A text editor allows you to write your programs. For Windows, WinSCP has an adequate text editor built in. For the Macintosh, TextWrangler works well. You are welcome to use other text editors, but a few features are important:
    1. The editor should display line numbers, since your compiler will use them to report errors.
    2. The editor should be able to save files with Unix line endings. Linux utilities expect each line to end wih a single LF character, but many Windows editors end lines with two characters, CR and LF. (Ancient Mac programs ended lines with CR, but this practice ended with OS X). Voluminous compiler errors are generated by the extra CR's.
  2. A Secure File Transfer Protocol (SFTP) client allows you to save files on the . Ideally, the SFTP client is integrated with a text editor, so that you can edit files on your machine and each save immediately updates the remote copy. For Windows, a good free SFTP client is WinSCP. For the Macintosh, CyberDuck works well; install TextWrangler first, then use the CyberDuck preferences to edit text files with TextWrangler.
  3. An Secure SHell (SSH) client allows you to compile and run your programs, and observe the results. For Windows, PuTTY is a good free ssh client. For the Macintosh, Terminal is a built-in application that works fine.

Using Your Own Machine

These instructions assume that you will use the machine provided by your instructor. The is inexpensive, however, and you are welcome to copy these tutorials to your own machine and work on them there, so long as you follow the submission procedure outlined below. I kept an installation log as I set up the machine for this class, which you are welcome to browse for pointers.

What to Submit

These tutorials require you to experiment, observe, and reflect. It is assumed that you will work through the activities as you read these instructions. Unnumbered questions on the following pages are for your observation and reflection; do not turn in the answers. The questions in the section called Self Test are also just that; do not turn their answers in, either.

The section called Assignment describes the work to be turned in. The assignments are designated with the letters a, b, c, ... etc. You will be given a letter in class, for the one you will turn in. You are encouraged to do others, but you are only responsible for one.

Suppose you are to do assignment for a. for Tutorial 1. This tutorial uses the files t1m.cpp (C++) and t1.s (assembly), and you will be responsible for submitting both types of files as well. References

For brevity URL's for online sources are embedded in links. If you wish to read a URL, examine the link href=___ addresses. The references above the — are the first places to look for information, those below in alphabetical order contain more obscure details.
  1. Carl Hamacher et al., Computer Organization, 5th edition, McGraw-Hill (2002), ISBN 0072320869.
    There is a newer 6th edition out, but it is targeted for embedded systems programming. The 5th edition is better suited to a general computer organization course.
  2. ARM® Infocenter, ARM Limited (2007-2013).
    The source for all ARM technical publications.
  3. ARM Developer Suite Assembler Guide, Version 1.2, Document Number: DUI 0068B, ARM Limited (Nov. 2001).
    A guide to ARM assembly. The 2001 edition lists mnemonics now deprecated (e.g. floating-point), but still used by the GCC assembler.
  4. raspberrypi.org, The Raspberry Pi foundation (2012-).
    The authority for news and and updates on the computer and its components, and pointers to other resources.
  5. Application Note 33: Fixed Point Arithmetic on the ARM, Document number: DAI 0033A, ARM Limited(1996, 2007).
    For processors without a hardware divide instruction, this note offers a work-around.
  6. ARM1176JZF-S™ Technical Reference Manual, Document Number DDI 0301H, ARM Limited (2004-2009).
    The ARM 1176JZF-S chip is the processor version packaged in the . This document describes the processor, its coprocessors, the instruction sets available, and the pipelines implemented.
  7. ARM Architecture Reference Manual, ARMv7-A and ARMv7-R edition, Document Number: DDI 0406C.b, ARM Limited (Jul. 24, 2012).
    Documents the ARM Instruction set for the version implemented in the . The actual machine codes associated with each assembly mnemonic are described bit-by-bit. The old floating-point mnemonics are also mapped to their VFP equivalents.
  8. Alex Chadwick, Baking Pi - Operating Systems Development, Computer Laboratory, University of Cambridge (2012).
    This is a combined laboratory in ARM assembly and operating systems development.
  9. Bash Reference Manual, Free Software Foundation, Inc. (2008).
    The reference manual for the Bourne-again shell (bash), the command-line interpreter.
  10. BCM2835 ARM Peripherals, Broadcom, Inc. (2012).
    The Broadcom 2835 System-On-a-Chip includes the itself, the timers, the interrupt controller, and most of the rest of the circuitry required to run the . This document describes the parts of the chip accessible to programmers, and their relationship to the ARM processor included on the chip.
  11. COTSE - Expect Reference, Packetderm, LLC (2003).
    A reasonably complete and concise reference of Expect commands.
  12. Debugging with gdb,  Tenth Edition, for gdb (GDB) Version 7.5.50.20130215, Free Software Foundation, Inc. (1988-2013), http://sourceware.org/gdb/current/onlinedocs/gdb/ .
    Describes the workings of the gdb debugger, and explains how to write .gdbinit files.
  13. Expect and TCL mini reference manual, Inguza Technology AB (2008-2012).
    A quick reference for Expect and the TCL commands you need to use it.
  14. Don Libes, Exploring Expect, O'Reilly Media (1994), ISBN 9781565920903.
    A guide to Expect, written by the developer.
  15. FPU control word definitions. ARM VFP version, Free Software Foundation, Inc. (2009), distributed with GCC.
    To read its contents, enter cat /usr/include/fpu_control.h from the command prompt. This file defines the usable macros, masks, and return values for FPU floating-point exception handling on the . This is the file that your compiler reads, when it attempts to write executable code. Where the documentation for these values is incomplete and/or obsolete, include files are the authority of last resort.
  16. GNU `make', Free Software Foundation, Inc. (2008).
    The reference manual for 'make', the program used to compile and link these tutorials.
  17. David Thomas, Introduction to ARM, self-published (2006-2012), http://www.davespace.co.uk/arm/introduction-to-arm
    A short course on the ARM Architecture for C programmers.
  18. Linux man pages, self-published, http://linux.die.net/man/, (1996-).
    These pages document the most common shell commands, API calls, file formats, etc. in a typical Linux system.
  19. Local man pages: alarm, kill, pacman, perror, signal, signal.h, strerror, ... various publishers, dates.
    These pages are accessible by typing 'man ___' from the command prompt. If you're not sure the name of the command or function you want, try 'apropos ___' Each man page is installed and updated along with the software it documents, so man pages on your machine are usually more accurate than the documentation you find online.
  20. Procedure Call Standard for the ARM Architecture, Document number: IHI 0042E, ARM Limited (Jul. 24, 2012).
    Describes the registers used to call procedures, pass arguments and return values.
  21. Using as, Free Software Foundation, Inc. (1988-).
    Describes the GNU family of assemblers, and the directives common to all assembly languages. Within this site, the page entitled ARM Machine Directives describes the assembly directives specific to the ARM architecture.
